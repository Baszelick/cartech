# 🚗 CarTech

> Enterprise-система управления автомобильной площадкой дилерского центра.

CarTech предназначен для автоматизации процессов приемки автомобилей, хранения, предпродажной подготовки (ПСО), контроля аккумуляторов, выдачи автомобилей и аналитики склада.

Проект разрабатывается как pet-project с архитектурой, максимально приближенной к коммерческим Enterprise-приложениям.

---

# Основные возможности

## Прием автомобилей

- регистрация поступления автовозов;
- создание нескольких автомобилей за одно поступление;
- история поступлений;
- комментарии к поступлению и каждому автомобилю.

## Склад

- несколько локаций;
- несколько площадок внутри локаций;
- фильтрация автомобилей;
- поиск по VIN.

## Автомобили

- бренд;
- модель;
- цвет;
- VIN;
- статус автомобиля;
- дата поступления;
- история изменений.

## ПСО

- прохождение предпродажной подготовки;
- хранение даты выполнения;
- изменение статуса автомобиля.

## Контроль аккумуляторов

- журнал проверок;
- планирование следующей проверки;
- история всех измерений.

## Пользователи

- авторизация;
- роли;
- журнал действий.

# Принципы проекта

При разработке используются следующие правила.

- Чистая архитектура.
- Один метод = одна ответственность.
- DTO используются всегда.
- Prisma Entity никогда не возвращаются наружу.
- Бизнес-логика находится только в сервисах.
- Все изменения базы данных происходят через миграции Prisma.
- Все сложные операции выполняются внутри транзакций.
